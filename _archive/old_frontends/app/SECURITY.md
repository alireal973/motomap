# Security Policy

## Supported Versions

Security fixes are applied to the latest code on `main`.

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Use one of the following:

1. GitHub Security Advisory (preferred), if enabled on this repository.
2. Directly contact repository maintainers on GitHub and request a private channel.

When reporting, include:

- Affected component or file
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

We will acknowledge receipt as soon as possible and work on a fix timeline based
on severity and reproducibility.
